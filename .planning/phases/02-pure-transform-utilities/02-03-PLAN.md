and `color` (bidirectional hex/RGB parse → Hex/RGB/Tuple/HSL block + a gated

hand-rolling TZ math (D-01). color proves the hand-rolled hex/RGB/HSL math
(D-03) and that the locked color gate composes with a brand-new styled command.

Output: two working commands, epoch integration tests, a color trycmd snapshot
(deterministic block under NO_COLOR), in-module unit tests for the pure parse
functions, and registry edits.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| arg/stdin → epoch parse | Untrusted date/timestamp strings parsed by chrono; ambiguous/invalid must error cleanly. |
| arg/stdin → color parse | Untrusted hex/RGB strings parsed by hand-rolled math; out-of-range must error cleanly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Denial of Service (self) | epoch `parse_date` | mitigate | All chrono parse results and the `LocalResult` DST cases return `Result`/error — never `.unwrap()`; unknown formats `bail!` with a hint (D-12, V5, FOUND-05). |
| T-02-06 | Denial of Service (self) | color `parse_color` | mitigate | Hex/RGB parsing returns `Result`; out-of-range RGB components and malformed hex error to stderr (exit 1), no panic, no array index overflow (V5). |
| T-02-07 | Tampering (output integrity) | color swatch | mitigate | Swatch gated on `is_color_on()` (the single Phase-1 gate), so piped output is byte-identical minus ANSI — no parallel/background color path that would leave a blank line when stripped (D-10). |

Surface note: pure transforms, no network/auth/session/storage. The real risk is panic-on-bad-input (mitigated) and color-path divergence (mitigated by reusing the locked gate).
</threat_model>
