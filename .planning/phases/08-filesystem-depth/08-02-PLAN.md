<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI globs → matcher | User `--ignore`/`--exclude` globs compiled and matched against walked paths |
| Win32 FFI → du | `GetCompressedFileSizeW` over a user-supplied directory tree (read-only query) |
| Filesystem → renderer | Untrusted file/dir names rendered to stdout/JSON |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-02-GLOB | Tampering | du `--exclude` / tree `--ignore` glob scope | mitigate | globset/ignore anchored relative to the target root; a malformed glob is a clean `anyhow` error (exit 1), never a panic; no `..` escape because matching is on already-walked, root-rooted relative paths |
| T-8-02-GI | Information Disclosure | nested `.gitignore` over/under-matching | mitigate | Ancestor-chain matcher checked deepest-first (Pitfall 4); 3-level fixture is the primary test; default output unchanged (opt-in) |
| T-8-02-FFI | Denial of Service | unsafe `GetCompressedFileSizeW` call | mitigate | Wide-encoded null-terminated path, single localized `unsafe` block, `INVALID_FILE_SIZE` disambiguated via `GetLastError`; read-only query registers no OS state |
| T-8-02-NAN | Tampering | percentage divide-by-zero | mitigate | `total==0 → 0.0%` guard; percentage is a render-only value, NOT a JSON float (A2) — JSON never carries NaN |
| T-8-02-ANSI | Information Disclosure | band color / progress contaminating `--json` | mitigate | `is_color_on()` forced false under `--json`; `is_json_on()` fork FIRST; `json_no_ansi` test asserts no `0x1B` |
| T-8-02-DOS | Denial of Service | non-UTF-8 NTFS names under `--json` | mitigate | `to_string_lossy()` everywhere (D-4); never `to_str().unwrap()` |
</threat_model>
