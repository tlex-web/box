<objective>
Add the two non-destructive depth features to `dupes` and `bulk-rename`. Both insert small, pure, unit-testable transforms into the existing pipelines without touching their safety logic.

- **DUPE-V2-01:** `dupes` uses multi-stage hashing (size → partial → full BLAKE3) and is hardlink-aware (paths sharing one NTFS file-index are collapsed, never counted as wasted space).
- **RENM-V2-01:** `bulk-rename` gains case transforms (`--case upper|lower|title`) and sequential numbering (literal `{n}` token with `--number-width`/`--start`/`--step`).

Purpose: the partial-hash stage is a re-bucketing inserted between `collect_by_size` and the rayon full-hash; the hardlink collapse uses `GetFileInformationByHandle` from `windows 0.61` — NOT the nightly-only `std::os::windows::fs::MetadataExt::file_index()` (the RESEARCH HIGH-impact correction to STATE.md:113 / PITFALLS Pitfall 5). `bulk-rename`'s `{n}`/`--case` are post-passes applied to `new_name` over the SORTED plan BEFORE the unchanged collision/cycle/separator pre-flight, so the load-bearing safety detector is untouched.

Output: extended `dupes` and `bulk_rename` modules; extended `tests/dupes.rs` and `tests/bulk_rename.rs`. (Deps were added by 08-01; this plan does NOT touch Cargo.toml.)
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI regex/replacement → rename plan | User pattern + `{n}`/`--case` produce target names that must pass the abort-all pre-flight |
| Win32 FFI → dupes | `GetFileInformationByHandle` over user files (read-only handle query) |
| File contents → hasher | Untrusted bytes streamed (partial + full BLAKE3) for equality only — NOT a security boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-03-INJ | Tampering | `{n}`/`--case` producing a path-separator or `..` target | mitigate | Transform runs BEFORE the unchanged `preflight`/`injects` detector (refuses `/`,`\`,`..`,`.`,dots/spaces); a generated collision is caught by abort-all-before-any |
| T-8-03-REPRO | Tampering | non-reproducible `{n}` over walk order | mitigate | Sort the plan by source path BEFORE assigning the counter (Pitfall 7); test asserts identical output across two runs |
| T-8-03-HL | Spoofing | content-equality mistaken for same-inode | mitigate | Collapse by `(dwVolumeSerialNumber, fileIndex)` via `GetFileInformationByHandle` before wasted-space; hardlink alias never counted (Pitfall 6) |
| T-8-03-NIGHTLY | Tampering | nightly-only std `file_index()` failing to compile on stable | mitigate | Use the `windows 0.61` Win32 call exclusively (RESEARCH Pitfall 1, issue #63010 OPEN); no `MetadataExt::file_index` in the tree |
| T-8-03-FFI | Denial of Service | unsafe `GetFileInformationByHandle` | mitigate | Single localized wrapped `unsafe`; handle from an opened `File`; read-only query registers no state |
| T-8-03-DOS | Denial of Service | non-UTF-8 NTFS names under `--json` | mitigate | `to_string_lossy()` (D-4); never `to_str().unwrap()` |
</threat_model>
