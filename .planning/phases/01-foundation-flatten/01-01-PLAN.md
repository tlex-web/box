<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell → box process | Command-line arguments are the only untrusted input this plan handles; clap validates structure, main() maps failures to exit codes. |
| box process → user terminal/streams | Data must go to stdout, messages/errors to stderr — a stream-confusion bug breaks pipes for every downstream command. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information disclosure | main() stream routing | mitigate | Errors and the NotImplemented message print via `eprintln!`/`eprint!` only; `stub_exits_1_to_stderr` asserts stdout is empty on error (FOUND-03). |
| T-01-02 | Tampering (exit-code contract) | main() try_parse mapping | mitigate | Bare-box and badcmd both assert exit 2 (`bare_box_exits_2`, `badcmd_exits_2`); a regression that collapses all errors to 1 fails the suite (D-07/D-08). |
| T-01-03 | Denial of service (panic) | command dispatch | mitigate | No `unwrap`/`panic!` in dispatch paths; all error paths return `Err` mapped to exit 1 (FOUND-05). |
| T-01-SC | Tampering | cargo dependency installs | mitigate | All new crates were slopcheck-clean in RESEARCH § "Package Legitimacy Audit" (dunce + 6 dev crates, all `OK`); no `[ASSUMED]`/`[SUS]` packages → no blocking checkpoint required. |
</threat_model>
