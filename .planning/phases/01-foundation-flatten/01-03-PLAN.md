<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source tree → flatten walker | Untrusted directory contents: symlinks/junctions, reserved names, deep paths, hidden dirs all cross here. |
| flatten plan → output directory | Every write must be collision-safe (no silent overwrite) and confined to the output dir (no path injection via encoded names). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-DoS | Denial of service | output-inside-source copy loop | mitigate | Canonical containment guard (`out_root.starts_with(src_root)`, lowercased) aborts before any I/O; `out_inside_src_aborts` test asserts it (Pitfall 4). |
| T-03-symlink | Tampering / Elevation | symlink/junction traversal | mitigate | `follow_links(false)` + `path_is_symlink()` skip; `skips_symlinks` test asserts no loop and a skip count (Pitfall 8). |
| T-03-pathinject | Tampering | encoded collision name escaping out dir | mitigate | `encode_relative` replaces all separators with `_`; `rename::encode_no_separator` unit test proves no `\`,`/`,`..` survives. |
| T-03-reserved | Tampering / DoS | reserved device-name write (CON/NUL) | mitigate | `sanitize_reserved` appends `_`; `rename::sanitize_reserved` unit test covers every reserved class — file is renamed, not lost (Pitfall 7). |
| T-03-overwrite | Tampering (data loss) | merge into existing output dir | mitigate | `occupied` set seeded from `read_dir(out_root)` (D-14); `no_silent_overwrite` test asserts a pre-existing name is renamed, not clobbered. |
| T-03-silentloss | Information disclosure / integrity | deep (>260) path copy | mitigate | All copy I/O `.context(...)`-wrapped; `deep_path_no_silent_loss` asserts a loud error, never a silent drop (FOUND-06, Pitfall 5). |
</threat_model>
