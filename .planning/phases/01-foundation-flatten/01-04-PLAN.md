<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| install.ps1 → Windows registry (HKCU\Environment) | The script mutates OS-registered persistent state (user PATH) read by every future shell; a wrong write corrupts other apps. |
| install.ps1 → current PS7 session environment | The live `$env:Path` must be refreshed correctly without dropping System paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-pathcorrupt | Tampering | user PATH registry write | mitigate | Raw read via `DoNotExpandEnvironmentNames` + `ExpandString` write when `%` present prevents REG_EXPAND_SZ→REG_SZ downgrade that corrupts other apps' `%VARS%` (D-02, Pitfall 3). |
| T-04-dup | Tampering | idempotent PATH append | mitigate | Split-on-`;` + `-inotcontains` dedup (not substring `.Contains`); re-install reports "already present" — human-verify step 5 confirms no duplicate. |
| T-04-elevation | Elevation of privilege | install scope | accept→mitigate | User-scope HKCU only, no admin/UAC, never Machine scope (D-01/D-02, ASVS V4) — eliminates the elevation surface entirely. |
| T-04-sessionpath | Denial of service (usability) | session refresh | mitigate | `$env:Path` rebuilt from User ⊕ Machine so System32 etc. are not dropped from the live session (D-03, Pitfall 9); smoke test + human-verify step 2 confirm same-session `box`. |
| T-04-policy | (operational) | execution policy / SmartScreen | accept | Documented `RemoteSigned`/`Bypass` in README; not a code vuln, user-scope only (Pitfall 9). |
</threat_model>
