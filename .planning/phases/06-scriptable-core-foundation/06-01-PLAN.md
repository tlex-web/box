|----------|-------------|
| filesystem → process | `%APPDATA%\box\config.toml` is untrusted file input deserialized into `Config` |
| process → OS clipboard | `--clip` pushes the command's stdout payload to the shared Windows clipboard |
| process → stdout (JSON channel) | `--json` output is consumed by `ConvertFrom-Json` / downstream parsers as data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | config source location (`core::config::config_path`) | mitigate | Read config from user-scoped `%APPDATA%\box` ONLY via `dirs::config_dir()` — never CWD or an arbitrary env-specified path. An explicit CLI flag always overrides config (precedence chain). [SPINE-05, V4 Access Control] |
| T-06-02 | Denial of Service | malformed/non-UTF-8 config (`load`) | mitigate | Malformed → clean `BoxError::Config` → exit 2 (D-10), NEVER a panic; missing → silent `Config::default()`. `toml::from_str` + `deny_unknown_fields` rejects unknown keys. Config files are tiny — no unbounded read. [V5 Input Validation; Pitfall 4] |
| T-06-03 | Information Disclosure | `--clip` writing to the shared clipboard (`flush_clip`) | accept | `--clip` is explicit/opt-in; the spine NEVER auto-clips and flushes only after a SUCCESSFUL dispatch (a failed command never clobbers the clipboard). Per-command secret-clipboard caveats (passgen) are Phase 7. [V4; D-08] |
| T-06-04 | Tampering | ANSI/escape injection into the `--json` channel (`emit_json`) | mitigate | `emit_json` writes raw `serde_json` (which escapes control chars in string values), never `.green()`; `init_output` forces COLOR_ON=false under json/clip; no BOM (to_writer). [V5; Pitfall 1] |
| T-06-SC | Tampering | toml/dirs package installs | mitigate | Both packages slopcheck-clean and triple-cited (CLAUDE.md/STACK.md/D-1) — see 06-RESEARCH.md § Package Legitimacy Audit (both `[OK]`, mature, correct crates.io repos). No `[ASSUMED]`/`[SUS]`/`[SLOP]` package this phase → no blocking-human checkpoint required. |
</threat_model>
