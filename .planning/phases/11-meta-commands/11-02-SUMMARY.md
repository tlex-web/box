# Phase 11 Plan 02: box completions Meta-Command Summary

**`box completions <shell>` emits a static clap_complete script generated from the LIVE final `Cli` (`Cli::command()` via `CommandFactory`) — so it reflects every subcommand incl. `config`/`completions` and the global `--json`/`--clip`/`--no-color` + Phase-8/9/10 depth flags — to stdout only, prefixed with an inert `#`-comment registration header (D-10); `install.ps1` hints by default and idempotently registers the recipe in `$PROFILE` only under `-RegisterCompletions` (D-11). The last v2.0 meta-command.**

## Accomplishments
- `box completions <shell>` (CMP-01) — a required-positional `clap_complete::Shell` (bash|zsh|fish|powershell|elvish, the D-09 rustup pattern; a bad value is a clap exit-2 usage error and `--help` lists the shells for free). `run()` prints an inert `#`-comment registration header then `clap_complete::generate(self.shell, &mut Cli::command(), "box", &mut stdout)`.
- **SC3 live-Cli reflection** — the emitted script references `config`, `completions`, `weather`, and the global `--json`/`--clip` (proven by `script_reflects_live_cli`), because `CommandFactory` hands the complete argument tree — no stale hand-maintained list.
- **D-10 two-place recipe** — the registration one-liner lives in BOTH inert places: the generated script's `#`-comment header (`header_is_inert_comment`) AND the subcommand's `--help` long_about (`help_documents_registration_oneliner`).
- **D-10 purity** — stdout carries exactly the header + script: empty stderr (fires even under redirection), no ANSI, no BOM (`powershell_script_is_pure`, reusing the uuid `json_purity` byte checks).
- `install.ps1` (D-11) — a `param([switch]$RegisterCompletions)` first-statement block; the default path prints the registration hint and NEVER touches `$PROFILE`; `-RegisterCompletions` idempotently appends a `# box completions` sentinel block (guarded by `Select-String`), creating `$PROFILE` + parent dir if absent. Idempotency simulated against a temp profile (second run skips; sentinel count stays 1); `pwsh` AST parse clean.
- Full CLI/dispatch wiring: `pub mod completions`, the `Completions(CompletionsArgs)` variant after `Config`, the `Commands::Completions(args) => args.run()` dispatch arm, and the `completions` row added to the help snapshot (25 commands).
- `clap_complete = "4.6"` is the ONLY new crate (D-12) — resolved to a single `4.6.7`, no duplicate clap version.

## Decisions Made
- **Single-sourced recipe** — `pub const PS_REGISTER_ONELINER` in completions/mod.rs feeds the generated-script `#`-header; the cli.rs long_about repeats the SAME literal, so D-10's two inert places share one spelling.
- **Pure `write_header`** — factored out and unit-tested (PowerShell two-line recipe + generic non-PS provenance line) so the inert-comment contract is locked terminal-free; the `#[non_exhaustive]` `Shell` match uses an explicit `PowerShell` arm + a catch-all.
- **Live-command registration** — install.ps1 registers `box completions powershell | Out-String | Invoke-Expression` (regenerates each shell start) rather than dot-sourcing a saved script, so it auto-tracks future `Cli` changes across upgrades (D-11).
