<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg → category/language filter | `--category`/`--language` values are untrusted; validated by clap `ValueEnum` before the command runs |
| embedded corpus → output | `include_str!` content is compile-time trusted (no runtime file I/O) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-ENUM | Tampering | `--category`/`--language` value | mitigate | `ValueEnum` rejects any value outside the fixed variant set → clap exit-2 usage error listing valid values; no free-form string reaches a filter path. |
| T-10-02-IO | Tampering | corpus source | accept | Buckets are `include_str!`-embedded at compile time; there is NO runtime file read, so no path-traversal / TOCTOU surface (mirrors the existing fortune/roast contract). |
| T-10-02-EOL | Data integrity | new per-bucket data files | mitigate | Each new `src/data/{fortunes,roasts}/*.txt` is covered by a `.gitattributes eol=lf` glob (root-cause `\r` fix) and the loaders still `str::trim` defensively. |
</threat_model>
