<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg → cycle count / label | `--cycles` bounded by clap; `--label` is display-only text printed to the user's own terminal + toast |
| pomodoro → Win32 MessageBeep | unsafe FFI; only a compile-time sound constant crosses it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-BEEP | Elevation/Tampering | `MessageBeep` unsafe FFI | mitigate | `unsafe` block is minimal and documented; the only argument is a compile-time `MB_*` constant — NO untrusted input reaches the FFI. Best-effort (never fails the run). |
| T-10-03-CYCLES | Denial of Service | `--cycles <N>` | mitigate | Bound via `RangedU64ValueParser` (clap exit-2 on an absurd value), same pattern as the existing `minutes` field; `--loop` is intentionally unbounded but user-cancellable (Ctrl+C/q/Esc restores the terminal, exit 1). |
| T-10-03-LABEL | Tampering | `--label` into terminal/toast | accept | The label is the user's own argument echoed to their own terminal/toast; no privilege boundary is crossed. Printed via crossterm `Print` on the countdown line (no shell/eval). |
| T-10-03-JSON | Info disclosure | display-only SC4 | mitigate | `run()` never calls `is_json_on`/`emit_json`; a grep gate asserts pomodoro emits no JSON document under the global `--json` flag. |
</threat_model>
