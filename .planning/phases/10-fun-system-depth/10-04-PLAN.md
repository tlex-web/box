<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config.toml file → process | untrusted file content parsed into typed Config; malformed → exit 2, missing → default |
| user-controlled cache key → filesystem path | the logical cache key derives from a user-controlled location string; it must NOT reach the filename verbatim |
| cache file on disk → process | a cache file may be absent, stale, truncated, or attacker-touched; a read must never trust its shape |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04-TRAVERSAL | Tampering | cache entry filename | mitigate | The filename is `blake3-hex(key).json` — a fixed hex charset under the cache dir; the raw location (which may contain `..`, `/`, `\`) is NEVER interpolated into the path, so a hostile location cannot escape `%LOCALAPPDATA%\box\cache\` or overwrite an arbitrary file. |
| T-10-04-CACHE-PARSE | Denial of Service | `cache::get` read | mitigate | `get` parses tolerantly: any IO/parse error or an out-of-TTL entry returns `None` (a MISS) — never a panic or `Err`. A truncated/garbage/attacker-touched file degrades to a fresh fetch, never a crash or a trust of unvalidated JSON shape. |
| T-10-04-CONFIG | Tampering | nested Config parse | mitigate | `deny_unknown_fields` + typed `Option<Enum>` fields preserved: an unknown key or an invalid `units`/`default_algo` value is `BoxError::Config` → exit 2 (abort before dispatch); a missing file is silent default (SPINE-05 tolerance unchanged). |
| T-10-04-CACHE-DOS | Denial of Service | `cache::put` unbounded growth | accept | Entries are small hashed-key JSON blobs holding only forecast text; no auto-cleanup this phase (mirrors the accepted T-8-06-ACCUM manifest-accumulation rationale). |
</threat_model>
