<objective>
Ship `box dupes` end-to-end (DUPE-01): a content-duplicate finder that groups byte-size first, then content-hashes only within same-size groups (most unique files are never hashed), parallelizes the hash phase with rayon, emits deterministically-sorted groups of identical files, and reports a wasted-space summary — all strictly read-only. Reuses the BLAKE3 hashing infrastructure built by the `hash` slice (Plan 03-01) and the `human_size` formatter (Plan 03-02/03).

Purpose: DUPE-01 is the only command needing parallelism; building it after `hash` lets it reuse the verified `blake3::Hasher::update_reader` streaming path for content equality (D-13) rather than re-deriving it.
Output: a working read-only `box dupes` command and Wave-0 integration tests.
</objective>
