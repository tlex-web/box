| Boundary | Description |
|----------|-------------|
| filesystem → du | untrusted directory tree walked read-only and summed |
| user argv (`path`/`--top`/`--depth`) → du | untrusted path + numeric args |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Denial of Service | symlink loop during recursive sum | mitigate | `follow_links(false)` on every descendant walk (D-06); no symlink is followed. |
| T-03-10 | Denial of Service | dotted-root pruned to zero | mitigate | Reuse `core::fs::is_hidden` (exempts depth-0); never re-implement (RESEARCH Pitfall 7). |
| T-03-11 | Tampering / Info Disclosure | UNC `\\?\` leak from `std::fs::canonicalize` | mitigate | `core::fs::normalize_path` (dunce) on the target (FOUND-06). |
| T-03-12 | Repudiation / correctness | non-deterministic walk-order output | mitigate | `collect` → `sort_by (size desc, name asc)` before printing; tests use distinct-size fixtures so the order is total (RESEARCH Pitfall 6). |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependency is added by this plan (reuses `walkdir`/`owo-colors`/the promoted `human_size`, all vetted in RESEARCH.md § Package Legitimacy Audit). No install-time checkpoint required. |
