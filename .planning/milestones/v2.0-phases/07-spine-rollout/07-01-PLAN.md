## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI args → command | untrusted text/flags (base64 input, color string, epoch arg) cross into the command |
| decoded bytes → JSON string | base64 --decode produces arbitrary bytes that must not corrupt the JSON document |
| generated secret → OS clipboard | passgen --clip writes a password to shared Windows clipboard state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07a-01 | Tampering / DoS | base64 --decode --json (non-UTF-8 bytes) | mitigate | A1: emit decoded payload as a base64 string field; NEVER `String::from_utf8(...).unwrap()`. `json_decode_non_utf8` test is the regression backstop (Task 1). |
| T-07a-02 | Information Disclosure | passgen --clip (secret to clipboard) | accept | Opt-in only (user typed --clip); document the clipboard-history/cloud-sync risk in a doc comment. Do NOT auto-clip. No auto-disclosure path exists. |
| T-07a-03 | Tampering (terminal escape) | JSON string values (control-char / ANSI injection) | mitigate | serde_json escapes control chars on serialize; emit_json never styles. The `json_purity` no-0x1B assertion (all 6 commands) is the backstop. |
| T-07a-04 | Tampering | display-only commands accidentally emitting a malformed JSON doc | mitigate | SC4 parse-but-ignore: the five display-only commands never call emit_json; `display_only_omit_json` test asserts no JSON document on stdout (Task 3). |
| T-07a-SC | Tampering | npm/pip/cargo installs | accept | Phase 7 installs ZERO packages (serde/serde_json/arboard already present — confirmed in RESEARCH § Package Legitimacy Audit). No install task exists; legitimacy gate is moot. |
