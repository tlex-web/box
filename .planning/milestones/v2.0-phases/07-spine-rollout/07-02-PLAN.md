## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem path → JSON string | NTFS paths (possibly non-UTF-8) cross into JSON string fields |
| large u64 size → JSON number | byte sizes >2^53 may lose precision in JS consumers (PS7 is safe) |
| conflict/abort state → stdout | a failed run must not leak a partial document onto stdout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07b-01 | Tampering / DoS (panic) | non-UTF-8 NTFS path in du/tree/dupes/flatten/bulk-rename JSON | mitigate | `to_string_lossy()` on every path field (D-4); NEVER `to_str().unwrap()`. Codebase convention; the JSON path must match the human path. |
| T-07b-02 | Information Disclosure | bulk-rename --json abort path leaking the plan-with-conflicts to stdout | mitigate | A3: guard the conflict print behind `if !is_json_on()`; under --json stdout stays EMPTY, error → stderr, exit 1 (D-09). `json_abort_empty_stdout` test is the backstop. |
| T-07b-03 | Tampering | bare u64 size >2^53 precision loss in JS consumers | accept | D-3: bare u64 is the locked whole-spine rule; PS7 (Int64/BigInteger) handles it; document the JS caveat. No mitigation beyond documentation. |
| T-07b-04 | Tampering (terminal escape) | JSON string values (paths, names) carrying control chars | mitigate | serde_json escapes control chars on serialize; emit_json never styles; `json_purity` no-0x1B assertion across all 5 commands. |
| T-07b-SC | Tampering | npm/pip/cargo installs | accept | Phase 7 installs ZERO packages (confirmed RESEARCH § Package Legitimacy Audit). No install task; gate moot. |
