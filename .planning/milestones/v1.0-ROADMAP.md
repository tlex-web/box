## Milestone Summary

**Decimal Phases:** None (all five phases were sequential, no urgent insertions).

**Key Decisions:**

- Phase ordering by integration risk: pure transforms (Phase 2) before filesystem tools (Phase 3) before terminal visuals (Phase 4) before the riskiest Windows-API / network commands (Phase 5) â€” find architecture problems on `uuid`, not on `flatten`, and have 21 commands working before the hardest integrations are attempted.
- Phase 1 bundles Foundation + Flatten â€” the foundation is not deliverable without a real command; shipping both means Phase 1 ends installable and usable.
- Single binary, single clap-derive registry, `src/commands/<cmd>/mod.rs` per command, `RunCommand` trait, `~40-line main.rs` (parse + dispatch + exit code only). `src/core/` holds errors/output/fs/input shared infrastructure, grown into by its first live consumer.
- SHA-256 (not BLAKE3) is the `hash` default â€” the REQUIREMENTS.md HASH-01 binding contract; BLAKE3 available via `--algo blake3`, BLAKE3-as-default deferred to HASH-V2-01.
- `x86_64-pc-windows-msvc` target with `crt-static` â€” MinGW demoted to Tier 2 in Rust 1.88; MSVC required for arboard / winrt; static CRT makes the exe portable.
- Two CLAUDE.md crate recommendations overridden during execution: `qrcode` over qr2term (D-01, Phase 5), `tauri-winrt-notification 0.7.2` over winrt-notification 0.5 (D-09, Phase 5 â€” modern `windows 0.61`, compiled clean first try).

**Issues Resolved:**

- flatten silent-data-loss edge cases (Windows trailing dot/space truncation, non-ASCII case-fold collisions, unconditional copy) hardened post-review (CR-01/WR-01/WR-02, Phase 1).
- bulk-rename path-escape via `..`/`.`/whitespace-only targets folded into the rule-4 separator refusal (CR-01 BLOCKER, Phase 3) â€” closes a path-escape on the only destructive command.
- matrix raw-mode-stuck-terminal on setup failure (RawGuard armed before the fallible EnterAlternateScreen) (CR-01 BLOCKER, Phase 4).
- ascii `cols*src_h` u32 overflow on large images â†’ u64 + u16::MAX clamp (WR-02, Phase 4).
- pomodoro out-of-range minutes panic-abort (Instant overflow) â†’ clap RangedU64ValueParser, exit 2 (WR-01, Phase 5).
- winrt toast compile spike + pomodoro blocking-vs-non-blocking timer + weather default-unit-system open questions all resolved during Phase 5 execution.

**Issues Deferred:**

- v2 per-command differentiators tracked in REQUIREMENTS.md v2 section: HASH-V2-01 (BLAKE3 default, multi-file, `--json`), VIS-V2-01 (lolcat `--animate`/`--freq`, matrix color/charset, qr `--save`, ascii color/braille), DEV-V2-01, FUN-V2-01, SYS-V2-01, PASS-V2-01, and per-tool TREE/DU/DUPE/RENM/FLAT-V2 enhancements.
- Phase 1 code-review advisory follow-ups (01-REVIEW.md, non-blocking): install.ps1 PATH empty-segment + smoke-test-by-abspath (WR-03/WR-04); shared flatten render path between dry-run and real run (IN-02/IN-03).

**Technical Debt Incurred:**

- Minimal. The advisory follow-ups above are the only known carryover; no shortcuts were taken on safety (every destructive/abort path is snapshot-tested byte-for-byte unchanged), correctness, or test coverage (150 bin-unit + all integration green, clippy `--all-targets -D warnings` + fmt clean at close).
