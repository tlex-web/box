## v2.0 Requirements — all shipped

### Scriptable Output Spine (cross-cutting)

- [x] **SPINE-01**: `box <cmd> --json` emits exactly one well-formed JSON document on stdout — no BOM, no ANSI, no progress, `snake_case` fields. *(Established/contract-tested on `uuid`+`hash` pilots; `json_purity` frozen as the template.)* — **Validated**
- [x] **SPINE-02**: `--json` on every applicable value-producing command (16 commands). — **Validated (Phase 7)**
- [x] **SPINE-03**: `box <cmd> --clip` copies the primary result to the Windows clipboard (copy + print; "Copied to clipboard" to stderr, TTY-gated). — **Validated (Phase 6 pilots)**
- [x] **SPINE-04**: `--clip` on every single-textual-result command (8 commands: passgen, uuid, color, hash, base64, epoch, json, qr). — **Validated (Phase 7)**
- [x] **SPINE-05**: Config-file defaults resolve CLI > env > config > built-in; missing/malformed config never errors a normal command. — **Validated (Phase 6)**

### Meta-commands

- [x] **HASH-V2-02**: `hash` accepts multiple file arguments, coreutils `digest␣␣filename` per line, stderr progress for large inputs. — **Validated (Phase 8)**

## Out of Scope (v2) — reasoning preserved

| Feature | Reason |
|---------|--------|
| `box self-update` | Network + replacing a running `.exe` on Windows (file-lock hazard) = high integration risk. |
| Scoop / winget packaging | `install.ps1` remains the chosen install path. |
| Cross-platform (macOS / Linux) | Target remains Windows PowerShell 7. |
| NDJSON / JSON-Lines output | PS7 `ConvertFrom-Json` cannot consume it inline; one buffered document is house style. |
| Per-command bespoke JSON schemas | Inconsistency defeats the scriptable spine. |
| Interactive prompts in destructive commands | Breaks scriptability and the 0/1/2 exit contract. |
| Config interactive wizard / TUI | `box config` is flag-driven; a wizard is unscriptable. |
| `json --sort-keys` as the default | Breaks the v1 `preserve_order` contract; opt-in only. |
