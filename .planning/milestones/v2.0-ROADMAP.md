# Milestone v2.0: Toolbox → Toolkit

**Status:** ✅ SHIPPED 2026-07-14
**Phases:** 6–11
**Total Plans:** 23

## Overview

v2.0 deepened the shipped 23-command `box` binary into a scriptable PowerShell-7 toolkit — a *deepening, not a rewrite*. The settled v1 architecture (single crate, `RunCommand` trait, `is_color_on()`-gated styling, 0/1/2 exit contract) was preserved; v2 grafted a cross-cutting `--json`/`--clip`/config spine onto that base, proved it on the two cheapest commands (`uuid` + `hash`), rolled it mechanically across every applicable command, then layered per-command depth flags, and finished with the two meta-commands. Phases were ordered by integration risk per the v1 retrospective's #1 lesson — the only real architecture risk lived in the shared spine, so it was built once on `uuid`+`hash` before 21 commands adopted it. The breaking change (BLAKE3-default `hash`) co-shipped with config so the `hash.default_algo = "sha256"` escape hatch existed the moment the default flipped. Completions landed dead last so the generated PS7 script reflects every flag added along the way.

## Milestone Summary

**Decimal Phases:** None — v2.0 executed as six clean integer phases (6–11).

**Key Decisions:**

- Order phases by integration risk: spine first, then rollout, then depth, then meta (v1 retrospective lesson #1 — build the shared spine once on the 2 cheapest commands before 21 adopt it).
- BLAKE3-default (HASH-V2-01) + config resolver (SPINE-05) co-located in Phase 6 so the `hash.default_algo = "sha256"` escape hatch exists the moment the breaking default flips.
- 27 depth reqs split into 3 area-grouped phases (8 fs / 9 dev+visual / 10 fun+system); meta-command phase stays strictly last (Phase 11).
- 3 destructive depth flags each get their own plan + mandatory adversarial review (08-04 `flatten --move`, 08-05 `dupes --delete`, 08-06 `bulk-rename --backup`).
- Config: hand-roll `toml` 1.1.2 + `dirs` 6.0.0, `Option<T>` + `.or().or().unwrap_or()`; precedence CLI > env > config > builtin; missing/malformed file → defaults, never errors a normal command.
- `--json` house style: ONE document, no NDJSON, no BOM, no ANSI/progress on stdout (SPINE-01); per-command `json_purity` test is the regression backstop.
- `completions` (CMP-01) generated from the live final `Cli` — must include every Phase-8/9/10 flag.

**Issues Resolved:**

- Established a consistent scriptable `--json`/`--clip` spine across all 23 commands with a single house style.
- Flipped `hash` to BLAKE3-default without silently breaking stored SHA-256 baselines (verify length table untouched; D-05 diagnostic on 64-hex mismatch).
- Shipped three destructive filesystem flags safely (dry-run default, `--force`, abort-all-before-any, snapshot-the-tree tests, adversarial review).
- Fixed a pre-existing v1 bulk-rename Windows trailing-dot/reserved-name silent-clobber data-loss path (Phase 8 CR-01).
- Fixed a `lolcat --duration` `Instant` overflow that would bypass the terminal-restore RAII guard (Phase 9 BL-01).
- Closed env-tier parity + malformed-config decoupling gaps in the config meta-command (Phase 11 WR-01/WR-02).

**Issues Deferred:**

- 9 human-verify PS7 items persisted as UAT (clipboard round-trips, live progress bars, `du --on-disk` vs Explorer, pomodoro/weather live UX, ASCII/color visual quality, PS7 tab-completion via PSReadLine) — recorded in STATE.md Deferred Items at close.
- `bulk-rename --undo` manifest replay (the `--backup` manifest is written and reconcilable; automated replay deferred — manual reverse documented in the module doc).

**Technical Debt Incurred:**

- Repo-wide pre-existing rustfmt drift — a `style: cargo fmt` root sweep deferred (newly-authored code is fmt-clean; the shipping gates use `clippy -D warnings`, which is green).
- Orphaned `core::config::resolve_algo` dead code (forward-compat `#[allow]`) flagged advisory by the Phase-11 code-review, unrelated to shipped behavior.
- v1 install.ps1 advisory follow-ups still carried (WR-03/WR-04 PATH empty-segment + smoke-test-by-abspath; IN-02/IN-03 shared flatten render path).
